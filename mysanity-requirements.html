Requirements Specification

MySanity AI Behavioural Intelligence Platform

Structured Business and System Requirements for an AI-powered mental health platform designed to enable proactive wellbeing through mood tracking, sleep analysis, journaling, and conversational AI (text + voice).

Business Requirements

  • BR1: Provide a unified mental health platform combining mood, sleep, journaling, and AI interaction.
  • BR2: Enable proactive mental health awareness through continuous behavioural tracking and insights.
  • BR3: Improve user engagement by delivering personalised, context-aware wellbeing feedback.
  • BR4: Reduce fragmentation of mental health tools into a single integrated ecosystem.
  • BR5: Support early identification of behavioural patterns linked to emotional wellbeing changes.

Functional Requirements

  • FR1: Users must be able to log daily mood states and emotional wellbeing entries.
  • FR2: System must capture and analyse sleep duration and sleep quality data.
  • FR3: Users must be able to create and store journal entries for emotional reflection.
  • FR4: System must support AI-powered conversational interaction via text input.
  • FR5: System must support AI conversational interaction via voice input and output.
  • FR6: System must generate personalised insights based on combined behavioural datasets.
  • FR7: System must provide a dashboard showing trends across mood, sleep, and behaviour.
  • FR8: System must store historical user data to enable long-term behavioural analysis.

Non-Functional Requirements

  • NFR1: AI responses must be generated within an acceptable low-latency threshold (≤ 2–3 seconds).
  • NFR2: All user data must be encrypted and securely stored due to sensitivity of mental health data.
  • NFR3: System must be scalable to support increasing behavioural data volume over time.
  • NFR4: Platform must maintain high availability to ensure continuous user access.
  • NFR5: Voice interaction must support reliable real-time processing with minimal delay.
  • NFR6: System must be designed with privacy-first principles and GDPR compliance in mind.

User Stories

  • As a user, I want to log my mood daily so I can understand my emotional patterns.
  • As a user, I want to track my sleep so I can identify how rest affects my mental health.
  • As a user, I want to chat with an AI assistant so I can receive personalised support.
  • As a user, I want voice interaction so I can engage naturally without typing.
  • As a user, I want insights across my data so I can improve my wellbeing proactively.

MVP Scope Definition

The MVP is designed to validate whether users engage with a unified AI-driven mental health platform that supports proactive wellbeing through behavioural tracking and conversational intelligence.

  • Core mood tracking functionality
  • Basic sleep logging and monitoring
  • AI conversational assistant (text-based)
  • Basic voice interaction capability
  • Initial behavioural insight generation engine
