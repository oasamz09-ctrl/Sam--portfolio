Requirements Specification

MySanity AI Behavioural Intelligence Platform

Structured definition of functional and non-functional requirements for an AI-powered wellbeing system integrating mood, sleep, and conversational intelligence.

Business Requirements

  • BR1: Provide users with a unified platform for tracking mood, sleep, and wellbeing data.
  • BR2: Improve user self-awareness through AI-driven behavioural insights.
  • BR3: Increase engagement through conversational AI (text and voice interaction).
  • BR4: Enable personalised recommendations based on behavioural patterns.
  • BR5: Reduce fragmentation of wellness tools into a single integrated system.

Functional Requirements

  • FR1: Users must be able to log mood entries daily.
  • FR2: System must record and analyse sleep duration and quality metrics.
  • FR3: AI system must support text-based conversational interaction.
  • FR4: AI system must support voice input and output interaction.
  • FR5: System must generate behavioural insights based on combined datasets.
  • FR6: Users must receive personalised wellbeing recommendations.
  • FR7: System must store historical user data for trend analysis.

Non-Functional Requirements

  • NFR1: System must respond to AI queries within 2 seconds under normal load.
  • NFR2: All user data must be securely stored and encrypted.
  • NFR3: Platform must support scalable data storage for behavioural history.
  • NFR4: System must ensure high availability (99% uptime target).
  • NFR5: Voice interaction must support low-latency processing.

User Stories

  • As a user, I want to log my mood so I can track emotional patterns over time.
  • As a user, I want to review my sleep data so I can understand my rest quality.
  • As a user, I want to talk to an AI assistant so I can get personalised wellbeing advice.
  • As a user, I want voice interaction so I can engage hands-free.
  • As a user, I want insights across my data so I can improve my lifestyle decisions.

MVP Scope Definition

The Minimum Viable Product (MVP) focuses on validating whether users engage with an integrated behavioural intelligence system.

  • Core mood tracking functionality
  • Basic sleep logging module
  • Text-based AI conversational assistant
  • Simple behavioural insight generation engine
